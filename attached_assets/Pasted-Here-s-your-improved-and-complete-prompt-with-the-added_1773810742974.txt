### Response Content Improvements

* Improve the **text response content** so it reads like a clear, well-structured explanation.

  * logical sentence flow

* **Do NOT repeat links inside the text bubble** if they are already provided as separate action buttons.
* Remove raw URLs or redundant link mentions from the text response.

### Action Button Optimization

* Limit the number of action buttons/links per response to a **maximum of 5 only**.
* If more than 5 links exist:

  * Prioritize the most relevant and useful ones
  * Group or omit less important links intelligently
* Ensure all displayed actions directly match the user’s intent.

  * Include appropriate **Font Awesome icons** alongside each item.
  * Present them in a **clean, well-formatted layout** (not inline clutter).
  * Use spacing or list-style formatting so they are easy to scan.
* Avoid plain, compressed text like: “React, Node, Django”
* Prefer structured output such as visually separated or listed items with icons.
* Ensure the design remains consistent with the existing chatbot UI.

### Behavior Guidelines

* Maintain consistency with the current chatbot UI, structure, and rendering logic.
* Do not introduce unnecessary complexity or entirely new systems unless required.
* Do not break existing response behavior for plain text responses.
* Keep everything reusable and aligned with the CMS/script-driven responses.
* Focus on **clarity first, actions second**.

### Goal

Transform chatbot responses into:

* **Well-written, properly formatted explanations (text bubble)**
